Sliders

Sliders are based on the Slick carousel libraries. More can be read on the website for specific documentation.

    Requirements

  • Parent container
    All slider/galleries need a parent container with a targetable class/ID. This is necessary for the scripts to iterate through child elements.
  • Greater than 1 child element
    Otherwise, it's not really a slider, just a static element.
  • Initialization script per slider
    For each slider on a page, an init function needs to be triggered. In the example below, slick is being run on any element of a class of slides insides an element with a class of slides. For the sake of best practices, IDs are only used where needed, and, thus, fairely specific class selectors are used: 
    $(".slider .slides").slick({
	dots: true,
	infinite: true,
	centerMode: false,
	slidesToShow: 1,
	slidesToScroll: 1,
	initialSlide: 0
});

Required Files

    CSS

  • section.css
    Container CSS file for all CSS related to non-article pages
  • story.css
    Container CSS file for all CSS related to non-article pages
  • rwd.css
    Container CSS file for all CSS related to the various breakpoints
  • reset.css
    CSS reset file, based entirely on Eric Meyer's reset
  • base.css
    The base set of CSS that renders the website at mobile.
  • header.css
    CSS related to the page header
  • slick.css
    CSS related to the Slick slider library used for galleries/sliders
  • slick-theme.css
    CSS related to the Slick slider library used for galleries/sliders
  • article.css
    Loaded only in story.css. Contains all baseline css for articles.
  • rail.css
    Covers all css to display the rail elements. NOTE: This only should be loaded at 980px and above.
  • slider.css
    CSS related to the various sliders. Due to some of them loading at even the lowest breakpoints, this should be loaded without any media queries.
  • footer.css
    Related to CSS for footer display at desktop (980px) and above

    JS

  • jquery.min.js
    Standard jquery library. Should be loaded in HEAD.
  • hamburger-menu.min.js
    Scripts related to the display/functionality of the Hamburger menu
  • login.js
    Handles scripts to display/animate login box
  • slick.min.js
    Scripts necessary for all slider/galleries to function. Only needs to be loaded on pages with one or more of those elements
  • breaking-news.min.js
    Functionality related to the disappearing of the breaking news box. NOTE: only necessary to load on pages where breaking box will be. Requirements initially had that as a piece of functionality exclusive to the homepage
  • header-scroll.min.js
    Handles the fade-in of the site logo on scroll.
  • text-change.min.js
    Font-resizing scripts. NOTE: Loads only on article pages

Page Files

    Pages

  • about-us.html
    Layout for About Us page
  • article.html
    Standard article page is defined as having a large image at top, no author photo, no column name above the byline.
  • article-alternate.html
    Alternate article page for columns and other special content. Features No large image, but instead an embedded image with the content, author byline includes author photo (or other art) and column name in large type face above byline.
  • contact-us.html
    Layout for the Contact Us page
  • index.html
    Standard homepage layout.
  • index-feature.html
    Shows an alternate top of page treatment. This is achieved by adding a "feature" class to the existing "lede" classed element.
  • serp.html
    A basic SERP page to replace existing search results. Largely laid out as currently displayed, but with some changes needed to work in the new design.
  • topic-page.html
    Standard Topic page layout.
